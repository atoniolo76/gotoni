## GPU Snapshot Development Roadmap

### Phase 1: Core Infrastructure ✅
- Provider abstraction (Lambda Labs, RunPod)
- SSH key management
- Basic instance launch/termination
- GPU type definitions with CUDA compatibility

### Phase 2: Container Support ✅
- Docker image specification in launch requests
- CUDA version extraction from Dockerfiles (archived)
- GPU compatibility mapping based on CUDA versions (NVIDIA docs)
- Resource estimation (VRAM, RAM) from libraries (archived)
- YAML config format for instance requirements
- CLI analyzer tool for config analysis and GPU compatibility checking

### Phase 3: Configuration System ✅
- YAML config format for instance requirements
- Automatic Dockerfile analysis
- Instance type recommendation engine
- Cross-provider compatibility checking

### Phase 4: CLI Tools ✅
- Repository analysis tool (`cmd/analyzer`)
- Auto-configuration generation from Dockerfiles
- Interactive setup wizard (planned)
- Batch processing capabilities (planned)

### Phase 5: Advanced Features
- Cost optimization
- Multi-instance orchestration
- Monitoring and metrics
- CI/CD integration
